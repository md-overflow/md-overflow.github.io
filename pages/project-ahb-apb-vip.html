01

Overview

This project implements a complete UVM-based Verification IP (VIP) for an AHB-to-APB bridge — a hardware design that translates transactions from the high-bandwidth AMBA High-performance Bus (AHB) to the simpler, lower-power Advanced Peripheral Bus (APB). The bridge interfaces an AHB master (typically a processor) with up to 4 APB peripheral slaves, each mapped to a 1 KB address window.

The VIP is structured around two independent UVM agents — an AHB Master Agent that generates and drives stimulus onto the AHB interface, and an APB Slave Agent that responds to and monitors APB transactions. A central scoreboard performs byte-accurate end-to-end checking across all transfer sizes, while functional covergroups on both bus sides measure verification completeness. The entire environment targets both Mentor QuestaSim and Synopsys VCS via a unified Makefile regression flow.

Methodology
UVM 1.2
Full class-based environment with factory, configuration objects, and TLM analysis ports throughout.
Bus Protocol
AMBA AHB → APB
AHB master side drives SINGLE, INCR, and WRAP burst types. APB side covers 4 peripheral slaves via one-hot Pselx.
Stimulus
Constrained-Random
Address aligned to transfer size, burst length constrained per type, slave address ranges encoded as constraints.
Bus Width
Configurable
Default 32-bit. Reconfigurable to 64 / 128 / 256 / 512 / 1024-bit via a single macro in definitions.v.
02

Architecture

The environment sits above the DUT, which itself is composed of three RTL sub-modules: ahb_slave.v decodes incoming AHB transactions and generates a valid signal; apb_controller.v runs the three-state FSM (IDLE → SETUP → ACCESS); and apb_interface.v drives the final APB signals to whichever of the 4 peripheral slaves is selected by the decoded address.

03

Verification Components

Both agents run in active mode. The AHB master agent generates constrained-random ahb_xtn sequence items — with address aligned to transfer size, burst length auto-constrained per type, and address values constrained to valid slave windows. The APB slave agent simultaneously responds to APB transactions and passes observed transfers to the scoreboard for comparison.

ahb_xtn — Key Constraints
AHB Transaction
Haddr constrained to slave ranges · Hsize: byte / halfword / word · Htrans: IDLE / BUSY / NONSEQ / SEQ · Hburst: SINGLE, INCR4/8/16, WRAP4/8/16 · address naturally aligned to Hsize
Scoreboard
Byte-Accurate Checking
Collects from both monitors via TLM FIFOs. Compares per-byte lane based on Haddr[1:0] and Hsize — byte, halfword, or full 32-bit word. Tracks pass / fail counts.
ahb_cg — AHB Coverage
AHB Covergroup
Coverpoints: HADDR (4 slave bins) · HWRITE (read / write) · HSIZE (byte / halfword / word) · Full cross: HADDR × HWRITE × HSIZE
apb_cg — APB Coverage
APB Covergroup
Coverpoints: PADDR (4 slave bins) · PWRITE (read / write) · PSELX (one-hot select bins) · Full cross: PADDR × PWRITE × PSELX
04

Slave Address Map

Each of the 4 APB slaves is mapped to a 1 KB address window. The AHB address constraints in ahb_xtn are directly derived from these ranges.

Slave Address Range Pselx Bit Window Size
Slave 10x8000_0000 – 0x8000_03FF[0]1 KB
Slave 20x8400_0000 – 0x8400_03FF[1]1 KB
Slave 30x8800_0000 – 0x8800_03FF[2]1 KB
Slave 40x8C00_0000 – 0x8C00_03FF[3]1 KB
05

Test Suite

All three tests extend ahbtoapb_base_test and are defined in test/ahbtoapb_test_lib.sv. Each test controls stimulus via the no_of_trans variable in its sequence and runs with +sv_seed random for non-deterministic stimulus by default.

Test 01
Single Write / Read
single_write_read_test
Drives NONSEQ single-beat write transactions followed by single reads across all 4 slave address ranges. Baseline functional check.
Test 02
INCR Burst Write / Read
incr_write_read_test
Drives incrementing burst writes (INCR4 / INCR8 / INCR16) then reads. Validates sequential address incrementing and multi-beat scoreboard checking.
Test 03
WRAP Burst Write / Read
wrap_write_read_test
Drives wrapping burst writes (WRAP4 / WRAP8 / WRAP16) then reads. Validates address wrap-around behaviour at burst boundaries.

07

Running Simulations

Navigate to the sim/ directory first. The Makefile supports both Mentor QuestaSim and Synopsys VCS — switch between them by editing the SIMULATOR variable at the top of the Makefile.

make regressClean, compile, run all 3 tests, merge coverage, open report.
make run_testCompile & run single_write_read_test.
make run_test1Compile & run incr_write_read_test.
make run_test2Compile & run wrap_write_read_test.
make view_wave1Open waveform for single write/read test.
make reportMerge coverage databases and generate HTML report.
make covOpen merged HTML coverage report in browser.
make cleanRemove all generated logs, databases, and intermediate files.
